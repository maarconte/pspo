## 2024-03-20 - Memoizing @tanstack/react-table configurations
**Learning:** For `@tanstack/react-table` components like `TableQuestions.tsx`, failing to memoize complex configurations (`columns`, `filterFns`) causes the table instance to recreate its internal processing pipeline on every render (e.g., when a user checks a single box or opens a modal). This can lead to significant lag on large datasets, as the table unnecessarily rebuilds column definitions and re-evaluates all filters.
**Action:** Always wrap `columns`, `filterFns`, and any column action handlers (e.g., `handleSelectQuestion`) in `useMemo` and `useCallback` respectively. Ensure variables that don't depend on component state (like static option arrays) are moved outside the component or helper function so they are instantiated only once.
