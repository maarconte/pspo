## 2024-03-12 - useMutation Un-memoization and Table Optimization
**Learning:** `useMutation` from `@tanstack/react-query` returns a new object instance on every render. Functions derived from it (like `handleDelete`, `handleUpdate`) will break memoization in consumer components if not explicitly memoized wrapping `mutateAsync`. Additionally, `@tanstack/react-table` (v8+) implementations require strict memoization of `columns`, `filterFns`, and handlers, otherwise the table instance recreates its internal pipelines on every render.
**Action:** When creating custom hooks around `useMutation`, always wrap the exposed handler functions in `React.useCallback(async () => mutateAsync(), [mutateAsync])` to maintain reference stability. When configuring `@tanstack/react-table`, always `useMemo` for static configuration objects.
